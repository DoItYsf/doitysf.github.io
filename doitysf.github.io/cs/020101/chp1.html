chp1.计算机概要

1.1 计算机应用分类

  • 桌面计算机：典型代表为个人计算机，桌面计算机强调单用户提供良好的性能，价格低廉，通常运行第三方软件。
  • 服务器：通常借助网络访问，服务器面向大型用户，可以执行单个复杂应用，也可以处理大量的简单作业，如大型Web服务器。这些应用通常基于其他来源的软件，并且往往为了特别的需要而加以修改或定制。制造技术与桌面计算机差不多，但能够提供更强的计算或I/O能力。服务器的设计通常强调可靠性，因为它的当机开小比单用户的桌面计算机大的多。
  • 嵌入式计算机：数量最多，应用和性能范围广泛。嵌入式计算系统的设计目标是运行单一应用程序或者一组相关的应用程序，并且通常硬件集成在一起以单一系统的方式一并交付用户。因此，尽管嵌入式计算机的数量庞大，还是有很多用户从来没有意识到他们正在使用计算机。

程序性能的影响因素：

硬件或软件部分 如何影响性能
算法 决定源代码的行数和执行I/O操作的数量
编程语言、编译程序和体系结构 决定每行代码对应机器指令的数量
处理器和存储器系统 决定指令执行的速度快慢
I/O系统（硬件和操作系统） 据诶的嗯I/O操作执行速度的快慢

系统软件有很多中，但有两种对于现代计算机系统是必需的：操作系统和编译程序
操作系统是用户程序和硬件之间的接口，为用户提供各种服务和监控功能。其最重要的作用是：

  • 处理基本的输入和输出操作
  • 分配外存和内存
  • 为多个应用程序提供共享计算机资源的服务

编译系统完成另一项重要功能：把高级语言编写的程序翻译成硬件能执行的指令。这个翻译过程相当复杂。
使用高级编程语言的好处：

  1. 可以是程序员用更自然的语言来思考，用英文和代数符号来表示，形成的程序开起来更像文字而不是密码表。而且高级程序语言的可以按照用途设计：Fortran为科学计算设计，Cobol为商业数据操作设计，Lisp为符号操作设计，等等。
  2. 高级语言提高了程序元生产效率。编程语言使用较少的行数即可表示出设计用意。
  3. 采用高级语言编写程序提高了程序相对于计算机的独立性，因为编译程序和汇编程序能够把高级语言程序翻译成任何计算机的二进制元指令。

1.3 硬件概念入门

任何计算机的基础硬件都要完成相同的基本功能：输入数据、输出数据、处理数据和存储数据。

组成计算机的五个典型部件是输入、输出、存储器、运算器和控制器，气候总最后两个部件通常合称为处理器。该组成与硬件技术无关，你总能够把任何计算机中的任何部件归于这五种之一。

1.3.1 光电鼠标

光电鼠标实际上是一个小型的光处理器，使用LED提供光源，带有一个极小的黑白照相机和简单的光处理器。当LED照亮鼠标第小的平面时，照相机以每秒1500次的采样频率将拍摄的照片连续的输送给光处理器。光处理器通过对比照片就可以判定鼠标移动的方向和距离。

1.3.2 显示器

如今的LCD显示器采用一种动态矩阵显示技术，每个像素都有一个三极管精确控制电流，使图像更清晰。在彩色有元矩阵中，还有一个红-绿-蓝屏决定三种颜色分量的强度，每个点需要三个三极管开关。
图像有像素矩阵组成，可以由位图表示。根据屏幕大小和分辨率，显示矩阵大小的范围很大。彩色显示器上每色用8为表示，每个像素用24为表示，可以表示几百万种不同的颜色。
支持图像的计算机硬件主要是光栅刷新缓冲或帧缓冲，用于保存位图。屏幕上的图像实际上保存在帧缓冲中，每个元素的位元模式以指定的刷新速率诸葛独处并发送到显示器上。

1.3.3 主板

主板上的小长方块是集成电路，俗称芯片。主板由三部分组成：连接前面提到的各种I/O设备的电路、内存和处理器。

1.3.4 内存

内存是程序运行时的存储空间，同时也用于保存程序在运行时所使用的数据。内存由多个DRAM芯片组成，被用来承载程序的指令和数据。与串行访问内存不同的是，无论数据存储在什么位置，DRAM访问内存所需的时间基本相同。

1.3.5 处理器

处理器是主板上最活跃的部分。他严格按照程序的指令执行，将数字相加测试结果，并按照结果发出控制信号使I/O设备作出动作。有时候人们把处理器成为中央处理器，即CPU。
处理器从逻辑上包括两个主要部分：数据通路和控制器，分别相当于处理器的肌肉和大脑。数据通路负责完成酸航速运算，控制器负责指导数据通路、存储器和I/O数倍按照程序的指令正确执行。
在处理器内部使用的是另一种存储器——缓存。缓存是一种小而快的存储器，一般作为DRAM的缓冲。cache采用的是另一种存储技术，成为静态随机访问存储器（SRAM），其速度更快且不那么密集，因此价格更贵。

硬件和软件的描述有一个共同点：越深入，所展示的信息就越多。因此，将低层次的细节隐藏起来，在更高层次的描述中就可以采用更简洁的模型。使用分层或抽象的方法，是设计复杂计算机系统的主要技术。
最重要的抽象之一是硬件和底层软件之间的接口。该抽象被命名为计算机的指令集体系结构，或简称为体系结构。计算机体系结构包括了程序员正确编写二进制机器语言程序所需的一切东西，如指令、I/O设备等。一般来说，操作系统需要封装I/O操作、存储器分配和其他低级的系统功能细节，以便应用程序员无需在这些细节上分心。提供给应用程序员的基本指令集和操作系统接口合称为应用二进制接口（ABI）
无论硬件还是软件都可以分成多个层次，每个较低的层次把细节对上层隐藏起来。硬件设计者和软件设计者都用这种抽象原则来应对计算机体系的复杂性。计算机体系结构是抽象层次中的一个关键接口——硬件和地层软件之间的接口。这一抽象接口是的同一软件可以由成本不同、性能不同的实现方法来完成。

1.3.6 数据安全

计算机中的内存是易失性的。为了区分易失性存储器和非易失性存储器，我们将前者成为主存储器，后者称为二级存储器。DRAM在主存储器中占主导地位，磁盘在二级存储器中占主导地位。所有服务器和工作站中的非易失性存储器都是磁盘。闪存也是一种非易失性存储器，主要用于手机，目前在音乐播放器甚至笔记本电脑中正在逐步取代磁盘。
磁盘通常由多个盘片组成，盘片以每分钟5400～15000转的速度绕轴高速旋转。金属盘片两面涂有磁性材料，相似的材料用在录音带或录影带上。为了读写硬盘上的信息，一个装有小线圈的活动臂紧靠两边盘面，称为“读写头”。整个驱动内部密封，使读写头更加靠近盘面。
硬盘直径通常为1～3.5英寸。为了满足新产品需要硬盘直径越来越小。一般来说硬盘尺寸越大性能越高，尺寸越小单位价格越低，每GB的最佳成本不同。由于硬盘采用机械部件，因此访问速度远远低于DRAM，硬盘为5～20ms，而DRAM为50～70ns，比磁盘快约100，000倍。
磁盘和主存有三个主要差别：

  • 磁盘是非易失性的，因为它使用磁介质；
  • 磁盘的访问速度慢，因为它使用机械装置；
  • 磁盘每GB的价格相对较低，因为它容量大，价格适当。

除了光盘和闪存外，目前正在应用的外存技术：

  • 光盘（包括CD、DVD）是最常见的可移动外存。蓝光（BlueRay Disk，BD）是继DVD之后的下一代光盘格式标准。
  • 基于闪存的可移动存储卡，通常采用USB接口，用于文件传输。
  • 磁带，只能提供慢速串行传输，一只用于磁盘悲愤，现在常用冗余磁盘代替。

1.3.7 与其他计算机通信

联网的计算机由以下几个主要优点：

  • 通信：在计算机之间高速交换信息
  • 资源共享：有些I/O设备可以由网络上的计算机共享，不必每台计算机都配备。
  • 远距离访问：用户可以不必在计算机旁边，而是在很远的地方使用计算机。

根据传输速度以及信息传输距离，通信代价随之增长，网络传输的距离和性能是多种多样的。以太网的传输距离可以达到1000公里，传输速率可以达到10Gbps。近距离范围之内通信使用局域网（Local Area Network，LAN），跨州、跨省通信则使用广域网（Wide Area Network，WAN）。广域网可以支持万维网（World Wide Web）。

1.4 性能

1.4.1 性能的定义

作为个人计算机用户，对降低响应时间感兴趣。而数据中心感兴趣的往往是吞吐率。因此，在很多情形下，和关注吞吐率的服务器相比，我们需要对嵌入式以及台式计算机采用不同的应用程序作为测试基准和不同的性能度量标准。

响应时间（response time）：也叫执行时间（execution time），计算机完成某任务所需的总时间，包括硬盘访问、内存访问、I/O活动、操作系统开销和CPU执行时间等。
吞吐率（throughput）：也叫带宽（bandwide），性能的另一种度量参数，表示单位时间内完成的任务数量。
在讨论计算机性能时，为了使性能最大化，我们希望人物的响应时间或执行时间最小化。对于某个计算机XXX，我们可以表达为：性能X=1/执行时间X性能_X = 1/执行时间_X性能X​=1/执行时间X​

1.4.2 性能的测量

程序的执行时间一般以秒为单位。然而时间可以用不同的方法来表示。对时间最直接的定义是向上的时钟时间，也叫响应周期、消逝时间等。这些术语均表示完成任务所需要的总时间（包括硬盘访问、内存访问、I/O操作和操作系统开销等）。
一个处理器同时运行几个程序是，系统可能更侧重于优化吞吐率，而不是最小化一个程序的响应时间。因此我们往往要把运行我们自己任务的时间与一般的响应时间区别开。我们可以使用CPU执行时间，简称CPU时间，它只表示在CPU上花费的时间，而不包括等待I/O或其他程序的时间。CPU时间还可以进一步分为用于用户程序的时间，称为用户CPU时间和操作系统为用户服务花去的CPU时间，称为系统CPU时间。要精确区分这两种CPU时间是困难的，因为通常难以分清哪些操作系统活动属于哪些程序，且不同系统也千差万别。
为了一致性，保持基于响应时间和基于CPU执行时间的性能差异。使用术语**系统性能（system performance）表示空载系统的响应时间，并用术语CPU性能（CPU performance）**表示用户CPU时间。

不同的应用关注计算机系统的不同方面。许多应用，特别是那些运行在服务器上的应用，主要关注I/O性能，所以此类应用程序既依赖硬件又依赖软件，对CPU执行时间最感兴趣。而在其他一些应用中，用户可能对吞吐率、响应时间或两者的复杂组合更为关注（例如最差相应时间下的最大吞吐率）。要改进一个程序的性能，必须明确性能的定义，然后通过测量程序执行时间来寻找性能瓶颈。

1.4.3 CPU性能及影响因素

下面的公式将最基本的尺度（时钟周期书和时钟周期时间）和CPU时间联系起来：
一个程序的CPU执行时间=一个程序的CPU时钟周期数×时钟周期时间一个程序的CPU执行时间=一个程序的CPU时钟周期数\times 时钟周期时间一个程序的CPU执行时间=一个程序的CPU时钟周期数×时钟周期时间
这个公式清除表示硬件设计这减少一个程序的CPU时钟周期数或减少时钟周期时间就能改进性能。由于时钟频率和时钟周期时间互为倒数，因此有：
一个程序的CPU执行时间=一个程序的CPU时钟周期数/时钟频率一个程序的CPU执行时间=一个程序的CPU时钟周期数/时钟频率一个程序的CPU执行时间=一个程序的CPU时钟周期数/时钟频率
由此可见，提高时钟频率也能改进性能。

1.4.4 指令的性能

由于计算机是通过执行指令来运行程序的，因此执行时间一定依赖于程序中的指令书。一种考虑执行时间的方法是，执行时间等于执行的指令数乘以每条指令的平均时间。因此一个程序需要的时钟周期数可以写为：
CPU时钟周期数=程序的指令数×每条指令的平均时钟周期数CPU时钟周期数=程序的指令数\times 每条指令的平均时钟周期数CPU时钟周期数=程序的指令数×每条指令的平均时钟周期数

CPI（clock cycle per instruction）：每条指令的时钟周期数，表示执行某程序或程序片段时每条指令所需的时钟周期平均数。

1.4.5 经典CPU性能公式

根据前面的定义，我们已经可以用指令书、CPI的时钟周期时间来写出基本性能公式：
CPU时间=指令数×CPI×时钟周期时间CPU时间 = 指令数\times CPI\times 时钟周期时间CPU时间=指令数×CPI×时钟周期时间
或
CPU时间=指令数×CPI/时钟频率CPU时间 = 指令数\times CPI/时钟频率CPU时间=指令数×CPI/时钟频率
这个公式将性能分解为三个关键因素。我们可以根据这些公式来比较不同的实现方案或评估某个设计的替代方案。

程序执行时间（单位：秒）：执行时间=秒/程序=指令数/程序×时钟周期数/指令×秒/时钟周期执行时间 = 秒/程序= 指令数/程序\times 时钟周期数/指令\times 秒/时钟周期执行时间=秒/程序=指令数/程序×时钟周期数/指令×秒/时钟周期
唯一能够被完全可靠测量的计算机性能指标是时间。例如，对指令集减少指令数目的改进可能降低时钟周期时间或提高CPI，从而抵消了改进的效果。

影响CPU性能公式中各种因素的原因：

硬件或软件指标 影响什么 如何影响
算法 指令数，CPI 算法决定源程序执行指令的数目，从而也决定了CPU执行指令的数目。算法也可能通过使用较快或较慢的指令影响CPI。例如，当算法使用更多的浮点运算时，将会导致CPI增大。
编程语言 指令数，CPI 编程语言显然会影响指令数，因为编程语言中的语句必须被翻译成指令，从而决定了指令数。编程语言也会影响CPI，例如，Java语言对数据抽象进行了大量支持，将进行简介调用，需要使用CPI较高的指令。
编译程序 指令数，CPI 编译程序的效率，既影响指令数，又影响CPI。因为编译程序决定了最后生成的计算机指令，所以会以复杂的方式影响CPI。
指令集体系结构 指令数，时钟频率，CPI 指令集体系结构影响CPU性能的所有方面，因为它影响完成某功能所需的指令数、每条指令的周期数以及处理器的时钟频率。

1.5 功耗墙

占统治地位的集成电路技术是CMOS（互补型金属氧化半导体），其主要的功耗来源是动态功耗，即在晶体管开关过程中产生的功耗。动态功耗取决于每个晶体管的负载电容、工作电压和晶体管的开关频率：
功耗=负载电容×电压2×开关频率功耗=负载电容\times 电压^2\times 开关频率功耗=负载电容×电压2×开关频率
开关频率是时钟频率的函数，负载电容是连接到输出上的晶体管数量和工艺的函数，该函数决定了导线和晶体管的电容。
功耗是电压平方的函数，能够通过降低电压来大幅减少功耗，每次工艺更新换代时都会这样做。一般来说，每代电压降低约15%。

虽然动态功耗是CMOS功耗的主要来源，但静态功耗也是存在的。因为即使在晶体管关闭的情形下，还是有泄露电流存在。因此，增加晶体管数目，就会增加漏电功耗，即使这些晶体管总是关闭的。各种各样的设计和工艺创新被用来控制电流泄漏，但还是难以进一步降低电压。

1.6 单处理器转变至多处理器

为了更大的吞吐量，桌面和服务器公司都在单片微处理器中加入了多个处理器，而不再继续追求单个程序运行在单个处理器上的响应时间。

并行性对计算性能一直十分重要，但它往往是隐蔽的。流水线是一种漂亮的激素，通过指令重叠执行使程序更快的运行，这是指令级并行性的一个例子。在抽取了硬件的并行本质后，程序员或者编译程序可认为在硬件中指令是串行执行的。

1.7 制造以及AMD Opteron X4基准

芯片的制造是从硅开始的。用特殊的化学方法对硅添加某些材料，可以把其细微的区域转变为以下三种类型之一：

  • 良好的导电性（类似铜丝）
  • 良好的绝缘体（类似塑料）
  • 可控的导电体或绝缘体（类似开关）
    晶体管属于第三种，VLSI电路（超大规模集成电路）是由数亿个上述三种材料组合起来并封装在一起所制成的。
    集成电路的制造是从硅锭开始的，硅锭经切片机切成片厚度不超过0.1英寸的晶圆，这些晶圆经过20～40步化学加工最终产生晶体管、导体和绝缘体。

1.7.1 SPEC CPU 基准测试程序

用户日常使用的程序是用于评价新型计算机最完美的程序。所运行的一组程序集构成了工作负载。要评价两台计算机系统，只需简单比较工作负载在两台计算机上的运行时间。最常用的测试方法是使用一组专门用于测量性能的基准测试程序。这些测试程序形成负载，用户期望预测实际负载的性能。
SPEC（System Performance Evaluation Cooperative）建立了重点面向处理器性能的基准程序集。为了简化测试结果，SPEC决定使用单一数字来归纳所有整数基准程序。将被测计算机的执行时间标准化，即将被侧计算机的执行时间除以一个参考处理器的执行时间，结果称为SPECratio。SPECratio值越大表示性能越快（SPECratio是执行时间的倒数）。

使用SPECratio比较两台计算机时采用的是几何平均值，这样可以使得无论采用哪个计算机进行标准化都可以得到相同的相对值，公式为：nΠi=1n执行时间比i^n\sqrt{\Pi^n_{i=1}执行时间比_i}nΠi=1n​执行时间比i​​
执行时间比是执行时间按参照计算机进行标准化的结果。

1.8 谬误与陷阱

陷阱

在改进计算机的某方面时期望总性能的提升与改进大小成正比。

Amdahl定律：
改进后执行时间=受改进影响的执行时间/改进量 + 不受影响的执行时间改进后执行时间 = 受改进影响的执行时间/改进量\ +\ 不受影响的执行时间改进后执行时间=受改进影响的执行时间/改进量 + 不受影响的执行时间
当我们知道一些函数所消耗的时间以及其潜在的加速时，我们就可以使用Amdahl定律预测性能的提升，将Amdahl定律与CPU性能公式结合，是一种很方便的性能评价工具。

谬误

利用率低的计算机功耗低

服务器的工作负载是变化的，所以在低利用率的情况下功率很重要。对于部分服务器，即使利用率只有10%，也会消耗掉大约2/3的峰值功耗。

陷阱

用性能公式的一个子集去度量性能。

曾经有一种用**MIPS（每秒百万条指令）**取代时间去度量性能的方法。对于一个给定程序，MIPS表示为：
MIPS=指令数执行时间×106MIPS = \frac{指令数}{执行时间\times 10^6}MIPS=执行时间×106指令数​
MIPS是指令执行的速率，它规定了性能与执行时间成正比，越快的计算机具有越高的MIPS值。
但其实用MIPS作为度量性能的指标存在三个问题。

  • MIPS规定了指令的执行速率，但没有考虑指令的能力。我们没有办法用MIPS比较不同指令集的计算机，因为指令数不同。
  • 在同一计算机上，不同程序会有不同的MIPS，因而一台计算机不会只有一个MIPS值。将执行时间用指令数、CPI、时钟频率带入可得：MIPS=指令数指令数×CPI时钟频率×106=时钟频率CPI×106MIPS=\frac{指令数}{\frac{指令数\times CPI}{时钟频率}\times 10^6}=\frac{时钟频率}{CPI\times 10^6}MIPS=时钟频率指令数×CPI​×106指令数​=CPI×106时钟频率​
  • 如果一个新程序执行的指令数更多，但每条指令的执行速度更快，则MIPS的变化与性能无关。

1.9 小结

硬件和软件设计者都用分层的方法构建计算机系统，每个下层都对其上层隐藏本层的细节。这个抽象原理是理解当今计算机系统的基础，但这不代表设计者只要懂得抽象原理就足够了。许多重要的抽象层次是硬件和底层软件之间的接口，称为指令集体系结构。将指令集体系结构作为一个常量可以使其不同的实现方法能够运行同一软件。产生一个副效应：这些预先排除可能需要接口发生变化的那些革新结构。
有一个可靠的测定性能的方法——用实际程序的执行时间作为尺度。该执行时间与我们能够通过下面公式测量到的其他重要指标相关：
秒数/程序=(指令数/程序)×(时钟周期数/指令数)×(秒数/时钟周期数)秒数/程序 = (指令数/程序)\times (时钟周期数/指令数)\times (秒数/时钟周期数)秒数/程序=(指令数/程序)×(时钟周期数/指令数)×(秒数/时钟周期数)

执行时间是唯一有效且不可推翻的性能度量方法。人们曾提出其他度量方法，但均以失败告终。有些从一开始就没有反映执行时间，因而是无效的；还有一些只能在有限的条件下有效，超出了限制的条件则失效，或是没有清晰的说明有效性的限制条件。
